# 需求

帮我实现一个读取邮件中代表应用和异常次数的饼图，饼图中每条记录格式为：应用名：异常次数。从饼图中解析出应用名，并根据已有的应用与负责人联系方式的通讯录，每行格式为：应用名：手机号，给负责人发消息：清及时修复您的{应用名}的异常

# 目标

- 功能目标：基于上述需求，实现：帮我实现一个读取邮件中代表应用和异常次数的饼图，饼图中每条记录格式为：应用名：异常次数
- 非功能目标（可选）：通知及时、可追溯；数据解析可靠；

# 设计要点

- 模块划分：数据解析/读取模块；通讯录/负责人查询模块；消息/通知发送模块；应用与异常数据模块；
- 关键接口/数据结构：解析/读取接口；负责人查询接口；消息发送接口；
- 技术选型与约束：依现有项目技术栈（如 Java 8、Spring Boot）；外部依赖（邮件、短信等）需与现有基础设施一致。

# 风险与开放问题

- 邮件与图表/饼图格式依赖约定，需与上游一致。
- 通讯录格式需与现有一致。
- 通知渠道与鉴权需确认。
