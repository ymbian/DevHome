## Context

DevHome 前端为静态 HTML + 单一 `common.css`，已有侧栏、主内容区、按钮、表单、卡片等基础样式；部分页面（如 design.html）有内联样式。需求是在不改变功能与结构的前提下，让整站视觉更「科技、高端」，提升品牌感知。

## Goals / Non-Goals

**Goals:**

- 建立一套可复用的「科技感、高端感」视觉规范（配色、排版、圆角、阴影、渐变、轻量动效）。
- 在 `common.css` 中实现该规范，并覆盖侧栏、主区、按钮、卡片、表单、主题横幅等。
- 各页面（首页、设计页、技改、工作计划等）统一采用该风格，必要时做页面级微调。
- 保持无破坏性：不删除或重命名现有 class，仅增强样式；不引入新构建或新运行时依赖。

**Non-Goals:**

- 不重写 HTML 结构或路由；不新增前端框架或组件库。
- 不改变后端 API、认证或业务逻辑。
- 不做响应式大改或移动端专属设计（可在现有布局上做小优化）。

## Decisions

- **视觉基调**：深色/中性背景 + 高对比主色（蓝/青/紫渐变）+ 清晰层级（阴影、边框、留白），辅以轻微过渡与 hover 反馈，营造「专业工具」感。可选：玻璃拟态或细边框强调卡片与侧栏。
- **实现载体**：仅用 CSS（含 CSS 变量）。样式集中在 `common.css`，页面内联仅保留页面特有覆盖；不引入 Sass/Less 或 CSS-in-JS，避免构建复杂度。
- **动效**：仅使用 `transition` 与简单 `transform`（如 hover 抬升、焦点环），不做复杂动画或自动播放，保证性能与可访问性。
- **兼容**：不依赖尚未广泛支持的属性；若使用较新属性（如 `backdrop-filter`），提供降级（如纯色背景）。

## Risks / Trade-offs

- **风格主观性**：不同人对「科技/高端」理解不同。→ 在 design 中明确色板与组件示例，便于评审与迭代。
- **与现有内联样式冲突**：design.html 等页有内联 style。→ 优先用公共 class 覆盖；必要时在 common.css 中增加更高特异性或页面级 class，避免散落内联。
- **性能**：渐变与阴影过多可能增加重绘。→ 控制数量与范围，动效仅限过渡，避免大面积 box-shadow 动画。

## Migration Plan

- 在 feature 分支修改 `common.css` 与各 HTML 页；本地与 staging 验证所有页面。
- 发布时直接替换静态资源；无需数据迁移或后端发布。
- 回滚：还原 `common.css` 与相关 HTML 的修改即可。
