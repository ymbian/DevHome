## Context

当前平台在开发、技改、工作计划三个页面都展示相同的顶部主题文案，但视觉层级偏弱，无法体现首页主题感。已有实现已经将主题文案抽到公共样式类（`.site-theme`），因此本次优化重点是统一强化视觉表现，而不引入新的页面结构或后端改造。

## Goals / Non-Goals

**Goals:**
- 将顶部主题文案升级为“品牌头图”级视觉，显著增强第一屏吸引力。
- 主题文案居中显示，形成页面视觉焦点。
- 提升标题字号、字重、字距，建立主标题层级。
- 引入蓝紫渐变背景、阴影卡片、圆角和充足留白，提升质感。
- 保持三个核心页面主题样式一致。

**Non-Goals:**
- 不改动主题文案内容本身。
- 不新增后端接口、数据库字段或配置项。
- 不引入第三方 UI 框架或动画库。

## Decisions

### 1) 继续使用公共样式类统一主题头图
- **Decision**: 所有页面继续复用 `.site-theme`，仅在 `common.css` 集中升级样式。
- **Rationale**: 一处改动全站同步，避免样式漂移。
- **Alternative considered**: 每页单独定制样式。被拒绝，因为会造成维护成本和一致性风险。

### 2) 采用高对比蓝紫渐变卡片作为主题容器
- **Decision**: 使用蓝紫渐变背景 + 白色高对比文字 + 卡片阴影。
- **Rationale**: 满足“首页主题感”和“吸引力”目标，同时保持开发者场景的技术品牌风格。
- **Alternative considered**: 单色背景。被拒绝，因为视觉记忆点不足。

### 3) 响应式字体放大策略
- **Decision**: 主题字体采用 `clamp` 控制，在不同宽度下保持明显放大且不溢出。
- **Rationale**: 保证桌面端主标题气质，同时兼顾小屏可读性。
- **Alternative considered**: 固定大字号。被拒绝，因为小屏可能拥挤或换行异常。

## Risks / Trade-offs

- [风险] 渐变与阴影过强导致内容区喧宾夺主  
  → Mitigation: 控制阴影透明度和留白节奏，确保后续正文仍是二级焦点。

- [风险] 字号放大后在小屏出现多行换行  
  → Mitigation: 使用 `clamp` 与 `text-wrap: balance` 平衡换行。

- [权衡] 强视觉风格可能与部分极简偏好冲突  
  → Mitigation: 保持主题区域仅占顶部一块，避免全页面重度装饰。

## Migration Plan

1. 在 `common.css` 更新 `.site-theme` 样式（居中、放大、渐变、阴影、圆角、留白）。
2. 保持三个页面现有 `.site-theme` DOM，不新增结构。
3. 手工验证三个页面在常见分辨率下展示一致且可读。
4. 如出现视觉回归，可回滚 `common.css` 中 `.site-theme` 样式块。
