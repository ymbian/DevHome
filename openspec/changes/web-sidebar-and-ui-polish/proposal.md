## Why

当前开发模块与技改模块通过顶部文字链接切换，入口分散、缺乏统一导航；页面为朴素静态样式，可读性与辨识度不足。将两个模块收拢到左侧边栏并做 Web 美化，可提升导航清晰度与整体观感，便于后续扩展更多模块。

## What Changes

- 增加**左侧边栏**：固定展示「开发」「技改」等入口，点击切换主内容区路由（需求页 / 技改页），当前模块高亮。
- **主内容区**：仅右侧区域随路由切换，保留现有需求输入、技改列表、设计页等页面功能不变。
- **Web 美化**：统一字体、间距、配色与基础组件样式（按钮、表单、卡片、边栏），使界面更一致、易读；不改变现有 API 与业务流程。

## Capabilities

### New Capabilities

- `web-shell`: 全局外壳：左侧边栏导航（开发 / 技改）、主内容区占位、基础布局与响应式约束。
- `web-ui-polish`: 视觉与交互 polish：统一配色、字体、圆角、阴影、按钮/表单/列表等组件样式，与 shell 风格一致。

### Modified Capabilities

- （无：本次仅前端布局与样式，不改变既有能力规格。）

## Impact

- **代码**：`platform/src/main/resources/static/` 下 `index.html`、`tech.html`、`design.html` 及新增/修改的 CSS、少量 JS（导航与高亮逻辑）。
- **依赖**：无新增后端依赖；可选使用内联或独立 CSS 文件。
- **兼容**：URL 与现有链接（如 `/`、`/tech.html`、`/design.html?id=xxx`）保持可用，仅增加侧栏与样式，不破坏现有书签或集成。
