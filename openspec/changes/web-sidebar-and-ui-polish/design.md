## Context

平台当前为静态多页：`index.html`（需求）、`tech.html`（技改）、`design.html`（设计），通过顶部「开发」「技改」文字链接跳转，无统一外壳；样式为简单 system-ui、灰框，缺乏层次感。技术栈为纯 HTML/CSS/JS，无构建与框架，部署在 Spring Boot 静态资源下。

## Goals / Non-Goals

**Goals:**

- 提供全局外壳：左侧固定边栏 + 右侧主内容区；边栏展示「开发」「技改」入口，点击切换内容（或跳转现有 URL），当前页在边栏高亮。
- 主内容区沿用现有页面（需求、技改列表、设计页），仅嵌入 shell 或共享布局，不重写业务逻辑。
- 统一视觉：配色、字体、间距、圆角、阴影、按钮/表单/卡片/边栏组件样式一致，整体更易读、可辨识。

**Non-Goals:**

- 不引入 SPA 框架或构建工具；不改变现有 URL 与 API；不做后端改动。
- 不做移动端优先响应式（可预留基础 max-width/侧栏折叠思路，非必须）。

## Decisions

1. **Shell 实现方式**  
   **选择**：每个页面（index、tech、design）内嵌同一段 HTML 侧栏 + 主内容区结构，并引用共享 CSS（如 `common.css`）与少量共享 JS（高亮当前 nav item）。  
   **备选**：iframe 或服务端模板——增加复杂度且当前无服务端模板引擎，故不采用。

2. **路由与高亮**  
   **选择**：沿用现有多页与 URL（`/`、`/tech.html`、`/design.html?id=...`）；侧栏链接直接为 `<a href="/">`、`<a href="/tech.html">`；高亮通过 JS 根据 `location.pathname`（及 design 页的 pathname）设置当前项 class（如 `.active`）。  
   **备选**：单页 + hash 路由——与现有设计页带 `?id=` 的用法不一致，且需大改链接，故不采用。

3. **样式组织**  
   **选择**：单一共享 `common.css`（或 `shell.css` + `components.css`），包含布局（侧栏宽度、主区 margin）、配色变量（可选 CSS 变量）、按钮/表单/卡片/边栏样式；各页面保留少量页面级覆盖。  
   **备选**：完全内联——不利于复用与维护，故仅对极小覆盖使用内联。

4. **设计页与 shell**  
   **选择**：design 页也包含同一侧栏，便于从技改/开发切回其他模块；「返回」或面包屑可保留为「开发」或「技改」以回到列表。  
   **备选**：设计页无侧栏——会与「统一导航」目标不一致。

## Risks / Trade-offs

- **重复 HTML**：三页各自包含侧栏片段，后续增删 nav 需改三处 → 通过注释约定与简单结构最小化，或后续考虑服务端 include/前端模板。
- **未用构建**：无法做 CSS 变量或组件树优化 → 接受；用清晰命名与单文件 CSS 控制复杂度。
- **兼容**：旧书签与直接访问 `/design.html?id=xxx` 仍有效，仅增加侧栏与样式，无破坏性变更。

## Migration Plan

1. 新增 `common.css`（及可选 `shell.css`），定义布局与组件样式。
2. 在 index、tech、design 三页顶部插入统一侧栏 HTML，主内容包在指定容器内，并引入 `common.css` 与高亮脚本。
3. 调整各页既有样式，去掉与统一风格冲突的局部样式，改用 common 中的类。
4. 手工验证：从首页点侧栏到技改、到设计页、返回，确认高亮与链接正确；确认现有 API 与流程不变。
