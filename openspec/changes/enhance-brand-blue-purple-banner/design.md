## Context

平台三大页面共享 `.site-theme` 主题条，已具备居中与主标题基础，但在品牌蓝紫“炫感”和文字清晰度方面仍有优化空间。当前目标是进一步提升主题条视觉冲击力，同时确保阅读舒适度和跨页面一致性。

## Goals / Non-Goals

**Goals:**
- 主题条呈现更鲜明的品牌蓝紫风格。
- 保证主题文案在各种屏幕下清晰、可读、吸引。
- 维持页面主题感，不破坏现有信息架构。
- 统一开发、技改、工作计划三页样式。

**Non-Goals:**
- 不改动主题文案内容。
- 不新增后端能力或接口。
- 不引入动画框架和新依赖。

## Decisions

### 1) 样式集中在共享 CSS 统一治理
- **Decision**: 仅在 `common.css` 的 `.site-theme` 中升级品牌蓝紫样式。
- **Rationale**: 一处改动全站一致，避免页面级样式漂移。
- **Alternative considered**: 各页面单独覆盖。被拒绝，维护成本高且易失一致性。

### 2) 使用蓝紫渐变 + 边框高光 + 卡片阴影
- **Decision**: 采用更有层次的品牌蓝紫渐变，叠加轻边框高光与阴影。
- **Rationale**: 形成“更炫”的主题条，同时保留卡片边界感。
- **Alternative considered**: 纯色背景。被拒绝，因为视觉记忆点不足。

### 3) 文字可读性优先：高对比白字 + 阴影 + 响应式字号
- **Decision**: 使用白字高对比、适度文字阴影、`clamp` 响应式字体。
- **Rationale**: 在炫彩背景下保证文字清晰且有吸引力。
- **Alternative considered**: 透明浅字。被拒绝，存在可读性风险。

## Risks / Trade-offs

- [风险] 视觉太强影响正文关注  
  → Mitigation: 控制主题条高度和阴影透明度，维持正文层级。

- [风险] 不同显示器上颜色偏差  
  → Mitigation: 采用对比度更稳定的蓝紫色阶，保留足够文字亮度。

- [权衡] “更炫”与“稳重”存在风格平衡  
  → Mitigation: 不增加复杂动效，仅强化配色与排版。

## Migration Plan

1. 更新 `common.css` 中 `.site-theme` 品牌蓝紫样式。
2. 验证三大页面主题条一致渲染。
3. 校验文字可读性与视觉层级，必要时微调阴影与对比度。
4. 如有回归，回滚 `.site-theme` 样式块。
